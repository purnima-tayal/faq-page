Product

What is Nexastack?

Nexastack is a Multi-Cloud Platform to automate Infrastructure for Multi-Cloud as well as capable of deployment on Kubernetes.

What Tools are supported by Nexastack?

Currently, it supports Helm, Terraform, and Ansible.

Can I Use Nexastack as a Workflow Engine?

Workflow is a manifest file that contains a set of tasks that will be executed sequentially or parallel in Nexastack. It has a fully-featured workflow engine that is intended to use for Automation Frameworks only.

I already have my Automation Modules. Can I use them in Nexastack?

You can add those modules to your GitHub repository and create a workflow that can use the existing modules.

How can I run Terraform modules through Nexastack?

To automate Terraform modules through Nexastack, you first have to integrate your cloud credentials , nd you can create a workflow using visual editor according to your terraform modules.

How can I deploy Helm charts using Nexastack?

Users have to add the Kube config of the working cluster to the Infrastructure. Also, a user has to create a workflow according to the helm charts.

What are the different Source Code Management tools supported by Nexastack?

Nexastack is currently supporting GitHub. Bitbucket and GitLab are planned for Next Released.

Can I manage my Infrastructure in Multi-Cloud using the Nexastack?

Yes, Nexastack allows you to manage Infrastructure for GCP and AWS.

Can I use Nexastack to manage my Cloud-Native Application on Multi-Cloud Infrastructure?

You can deploy the application's Helm charts on Supported Cloud Providers in Multi-Cloud Architecture using workflow.

How much time does workflow take to run completely?

It depends on the no of tasks in the workflow and also the duration of these tasks.

Who will be able to add a workflow?

Any member of the project can create a workflow.

Which type of users can add an infrastructure?

Only Admin of the project and the owner of the workspace can add the Infrastructure.

What is a Registry?

A Registry is the collection of different modules of Ansible, Terraform, and Helm, and a user can add them to the workflow using web UI.

How to add a module in a workflow?

You can either add a module in GitHub or add it from the registry using web UI.

Can I add module code directly in the GitHub repository?

Yes, you can add modules directly to the GitHub repository.

Is it possible to add two or more admins in the workspace?

Yes, there can be multiple admins of a workspace.

Can I use my custom modules of Terraform, Helm, and Ansible?

Yes, you can add your custom modules to the GitHub repository.

Can I add a module to the registry?

No, you can't add modules to the registry at this moment. Nexastack will be supporting the Private Registry in the future.

Can I change the code of modules that were added from the registry module?

You can easily change them through GitHub IDE or on your local like any code and push it to IaC Repository.
